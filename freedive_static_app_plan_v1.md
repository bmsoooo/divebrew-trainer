## 0. 한 줄 정의

**divebrew trainer (가칭)** — "손목에서 시작하는 스태틱 훈련 코치". CO2/O2 테이블 훈련을 음성·햅틱으로 가이드하고, 심박·다이빙 기록을 자동으로 남기는 프리다이버용 훈련 앱.

**플랫폼 전략 (v1.2 확정)**: 웹앱(PWA) 선행 개발로 코어 검증 → 완성 후 **iOS 앱 최우선** 출시 → Apple Watch → Android는 후순위. Flutter 단일 코드베이스로 웹/iOS/Android를 모두 커버하므로, 웹 단계의 코드가 iOS로 그대로 승계된다.

---

## 1. 왜 만드는가 (문제 정의)

- 기존 스태틱 훈련 앱은 "폰 화면을 보면서" 훈련하는 구조다. 그러나 스태틱의 본질은 **눈 감고 이완**이므로, 폰을 보는 UX 자체가 훈련을 방해한다.
- 훈련 기록(숨참기 시간)과 생체 데이터(심박수), 실제 다이빙 로그(수심·수온)가 **세 개의 앱에 흩어져 있다.**
- 안전 고지가 형식적이다. 과호흡 금지·드라이 전용 원칙이 UX에 구조적으로 내장된 앱이 없다.

## 2. 경쟁 앱 분석 (스크린샷 기반)

| | Breath Hold – FreeDive | Freediving Ai Trainer | 본 앱의 차별화 |
|---|---|---|---|
| 훈련 테이블 | CO2 / O2 고정 구조 | CO2 / O2 / STATIC / 커스텀 라운드 편집 | 커스텀 편집 + **워치 단독 실행** |
| 가이드 | 음성 (한/영) | 화면 중심 | 음성 + **손목 햅틱** (눈 감고 훈련 가능) |
| 생체 데이터 | 없음 | 실시간 HR/SpO2 차트, 컨트랙션 기록 | 워치 심박 자동 기록 (외부 기기 불필요) |
| 기록 | 최고 기록 그래프, 핀치 줌 | 히스토리 상세, 메모 | 훈련 + **다이브 로그 통합** (Ultra 수심·수온 자동) |
| 커뮤니티 | 없음 | 로그 피드, 대회 정보 | v1 범위 제외 (후순위) |
| 안전 | 텍스트 고지 (물 밖 훈련용, 과호흡 금지) | 미확인 | **온보딩 동의 게이트 + 세션 시작 리마인더** 구조화 |

핵심 포지셔닝: 두 앱의 교집합(테이블 + 기록)은 기본으로 깔고, **"워치 네이티브 + 안전 내장"** 두 축으로 차별화한다. 소셜 피드·대회 정보는 v1에서 의도적으로 배제한다 (단순함 우선).

## 3. 타깃 사용자

1. **코어**: AIDA/PADI 자격 보유 프리다이버 — 주 2~3회 드라이 스태틱 훈련, 풀/바다 세션 병행
2. **입문자(프린이)**: 숨참기 1~2분대, 체계적 테이블이 뭔지 모르는 단계 — 프리셋 테이블 + 안전 교육이 온보딩 역할
3. **확장**: 호흡 명상·이완 목적 일반 사용자 (마케팅 관점 참고만, 기능 설계는 다이버 기준)

## 4. 기능 명세 (Phase별)

### Phase 1 — MVP: 웹앱 (Flutter Web, PWA)

**성공 기준: 모바일 브라우저에서 CO2/O2 훈련 1세션을 처음부터 끝까지 완료하고, 재방문 시 기록이 남아 있다.**

> 웹 단계 제약 (의도된 한계 — iOS 앱에서 해소):
> - 모바일 브라우저는 화면 꺼짐/백그라운드에서 타이머·오디오를 보장하지 못한다 → **Wake Lock API로 화면 유지가 기본 동작.** "화면을 켜둔 채 훈련"을 UX 전제로 명시
> - HealthKit·워치·수심 연동 불가 → 전부 Phase 3 이후로 이동
> - PWA 매니페스트 적용: 홈 화면 추가 시 앱처럼 실행

- **훈련 테이블**
  - CO2 테이블: 숨참기 고정, 회복 시간 점감 (프리셋 3종: 입문/중급/상급)
  - O2 테이블: 회복 고정, 숨참기 점증 (프리셋 3종)
  - 커스텀 테이블: 라운드별 breath/hold 초 편집, 라운드 추가·삭제, 총 훈련 시간 자동 계산, 저장·재사용
  - 개인 최고 기록(PB) 기반 테이블 자동 생성 — PB 입력 → 40~80% 구간으로 라운드 자동 산출
- **세션 실행**
  - 준비 호흡 → 숨참기 → 회복의 전 단계 타이머
  - 음성 가이드 (한국어/영어 — 웹은 Web Speech API, 앱은 flutter_tts로 추상화 계층 하나 두고 분기)
  - (웹) Wake Lock으로 세션 중 화면 유지 / (iOS 앱 단계) 백그라운드·화면 꺼짐 동작으로 승격
  - 세션 중 컨트랙션(횡격막 수축) 시점 탭 기록 (선택 기능)
- **기록**
  - 세션 히스토리: 날짜, 테이블 종류, 라운드별 결과, 메모
  - PB 추이 그래프 (CO2/O2/스태틱 분리 조회)
- **안전 (기능이자 브랜드)**
  - 최초 실행: 안전 동의 게이트 — "드라이 전용 · 과호흡 금지 · 혼자서도 앉아서 땅 위에서만 · 어지러우면 즉시 중단" 명시 동의 없이 진행 불가
  - 매 세션 시작 화면: 1줄 리마인더 로테이션 (훈계 톤이 아닌 1인칭 경험형 문구)
  - 준비 호흡 가이드는 과호흡을 유도하지 않는 템포로 설계 (분당 호흡수 상한 고정)
- **데이터**: 전부 로컬 저장 (웹 = drift WASM/IndexedDB, 앱 = SQLite — drift가 양쪽 지원). 계정·클라우드 없음. 단, 웹 로컬 저장은 브라우저 데이터 삭제 시 유실됨을 UI에 고지하고, JSON 내보내기/가져오기를 제공해 iOS 전환 시 기록 이관 경로 확보

### Phase 2 — iOS 앱 출시 (동일 코드베이스, 네이티브 승격)

**성공 기준: App Store 심사 통과 + 화면 꺼짐 상태에서 세션 완주 (웹의 핵심 제약 해소).**

- Flutter iOS 타깃 활성화 — 웹에서 검증된 화면·로직 그대로, 플랫폼 분기만 추가
- 백그라운드 타이머: 오디오 세션 유지 방식 → **실기기 화면 꺼짐 10분 세션 완주가 완료 판정**
- 음성 가이드 flutter_tts 전환, 로컬 알림 폴백
- 웹 기록 JSON 가져오기로 기존 사용자 데이터 이관
- Android 타깃은 이 시점에 활성화하지 않는다 (후순위 Phase 5 — 코드는 이미 호환되므로 QA·스토어 작업만 남는 상태 유지)

### Phase 3 — Apple Watch 컴패니언 (심박 + 손목 가이드)

**성공 기준: 폰을 가방에 넣은 채, 워치만 차고 눈 감고 1세션을 완료할 수 있다.**

- watchOS 네이티브 앱 (SwiftUI): 테이블 선택 → 세션 실행을 워치 단독으로
- 단계 전환을 **햅틱 패턴**으로 구분 (준비/홀드 시작/홀드 종료/세션 완료 — 각기 다른 진동)
- HKWorkoutSession으로 세션 중 심박수 연속 기록 → 세션 종료 후 폰 앱에서 심박 곡선 + 라운드 오버레이 차트
- WatchConnectivity로 폰 ↔ 워치 세션 데이터 동기화
- 제약 명시: SpO2 실시간 측정은 서드파티 API가 없으므로 미지원. 사후 HealthKit 샘플 조회만 참고 표시.

### Phase 4 — 다이브 로그

**성공 기준: 풀/바다 세션 1회의 수심·수온·시간이 자동 또는 30초 내 수동 입력으로 기록된다.**

- **자동 (Apple Watch Ultra 전용)**: CMWaterSubmersionManager로 수심(최대 40m)·수온·잠수 시간 자동 캡처 → 다이브 프로파일 저장
- **수동 (전 기종 + Android)**: 장소 유형(풀/바다), 최대 수심, 시간, 버디, 컨디션 메모 — 빠른 입력 폼
- 훈련 기록과 다이브 로그를 하나의 타임라인으로 통합 조회

### Phase 5 — 후순위 (검토만)

- **Android 앱 출시** (코드베이스는 이미 호환 — QA + Play Store 등록 작업)
- Wear OS(갤럭시 워치) 컴패니언 — 심박 기록 한정
- 클라우드 동기화 / 계정 (Supabase 또는 Firebase)
- 로그 공유 카드 (이미지 내보내기 — 인스타그램 공유용)
- 버디/커뮤니티 기능

## 5. 기술 아키텍처

### 스택 결정과 근거

| 계층 | 선택 | 근거 / 대안 |
|---|---|---|
| 웹 + 앱 | **Flutter** (웹 타깃 선행 → iOS → Android) | 단일 코드베이스가 웹/iOS/Android 전부 커버 → 웹 검증 코드가 iOS로 무손실 승계. 대안: React 웹앱(웹 완성도↑, iOS 단계에서 재작성 — 기각) |
| 워치 앱 | **SwiftUI 네이티브** (선택지 없음) | Flutter/RN은 watchOS 미지원. HealthKit·CoreMotion·수심 API 모두 네이티브 전용 |
| 폰↔워치 통신 | WatchConnectivity + 플랫폼 채널 | Flutter 쪽에서 MethodChannel로 브릿지 |
| 로컬 DB | drift | 웹(sqlite3 WASM/IndexedDB)·앱(SQLite) 동일 API. 오프라인 완결 |
| 차트 | fl_chart | 심박 곡선 + 라운드 오버레이 |
| 음성 | 추상화 계층 (웹: Web Speech API / 앱: flutter_tts) | MVP는 TTS, 녹음 음성 교체는 후속 |
| 생체 데이터 | HealthKit (iOS) / Health Connect (Android, 후순위) | 심박 기록·조회 |

### 플랫폼별 제약 (반드시 인지)

1. **수심·수온 자동 기록 = Apple Watch Ultra + watchOS 9 이상 전용.** 일반 애플워치·안드로이드에는 수심 센서가 없다 → 수동 로그가 기본 경로, 자동은 프리미엄 경험.
2. **SpO2 실시간 스트리밍 불가** (Apple 비공개 API). 경쟁 앱의 실시간 SpO2는 외부 BLE 옥시미터 연동으로 추정 — 필요 시 Phase 4에서 BLE 옥시미터 지원 검토.
3. **모바일 웹 백그라운드**: 화면 꺼짐 시 JS 타이머·오디오가 정지될 수 있음 → 웹 단계는 Wake Lock으로 화면 유지가 공식 UX. 백그라운드 완전 동작은 iOS 앱(Phase 2)의 핵심 승격 포인트.
4. **iOS 백그라운드 타이머**: 오디오 세션 유지 방식. **Android 백그라운드(Foreground Service)는 Phase 5에서.**
5. 워치 단독 세션은 폰 없이도 동작해야 하므로 워치 로컬 저장 → 연결 시 동기화 구조.

### 데이터 모델 (요약)

```
TrainingTable   id, name, type(CO2|O2|STATIC|CUSTOM), rounds[{breathSec, holdSec}]

DiveLog         id, date, siteType(POOL|SEA), maxDepthM, waterTempC?, durationSec,

```

## 6. 안전 설계 원칙 (요약 — 상세 문구는 별도 카피 문서로)

- 드라이 전용임을 앱 정체성 차원에서 명시. "물속·욕조에서 사용 금지"를 온보딩 동의 항목에 포함
- 과호흡 방지: 준비 호흡 가이드의 템포 상한 고정, "몰아쉬기 금지" 세션 리마인더
- 어지러움·이상 증상 시 즉시 중단 — 세션 중 항상 노출되는 중단 버튼 (확인 팝업 없이 1탭 종료)
- 다이빙 로그 쪽: 버디 필드를 기본 노출해 버디 원칙을 UX로 상기
- 톤: 훈계형이 아닌 1인칭 경험형 문구 (divebrew 안전 고지 원칙과 동일 철학)

## 7. 화면 구성 (MVP 기준 6개)

1. 온보딩 + 안전 동의 게이트
2. 홈 — 오늘의 훈련 제안 + PB 요약 + 빠른 시작
3. 테이블 목록/편집 (CO2·O2·커스텀)
4. 세션 실행 — 대형 타이머, 라운드 진행 표시, 중단 버튼 상시 노출
5. 세션 완료 — 결과 요약 + 메모 + 컨트랙션 타임라인
6. 히스토리 — 목록 + PB 그래프
